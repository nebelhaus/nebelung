# design-sync notes — nebelung

- This repo is NOT a component design system: no package.json, no npm build, no
  Storybook. It's a whiskers/tera theme repo (terminal-tool ports of the
  Nebelung palette).
- The sync is therefore **tokens-only and off-script**: `ds-bundle/` is
  generated by hand (see the generator flow below), not by the converter.
  No `_ds_bundle.js`, no components/, and deliberately no `_ds_sync.json`
  anchor — a re-sync re-verifies/re-uploads everything (6 files, trivial).
- Source of truth: `palette/nebelung.hex.json` (name → hex, no `#` prefix).
  The bundle exposes each as `--nebelung-<name>` plus semantic aliases
  (bg/fg/border/accent/status); accent = mauve, per repo convention.
- Bundle layout uploaded: `styles.css` (imports `tokens/nebelung.css`),
  `tokens/nebelung.{css,json}`, `README.md` (conventions + token index),
  `_preview/palette.html` (swatch card, registered via register_assets since
  there are no component cards).
- On palette changes: regenerate the four bundle files from the palette JSON,
  re-validate README token names against `tokens/nebelung.css`, re-upload the
  changed files, and re-write `_ds_needs_recompile` last.
- `ds-bundle/` is generated output, gitignored.
